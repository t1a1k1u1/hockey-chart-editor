## 1. 概要

### 1.1 目的

`chart.json` を GUI で作成・編集するデスクトップアプリケーション。
手動での JSON 編集に代わり、音楽を再生しながら視覚的にノーツを配置できる。

### 1.2 仕様の前提

| 項目 | 内容 |
|------|------|
| 実装プラットフォーム | Godot Engine 4.x（ゲーム本体とは別プロジェクト） |
| 操作入力 | マウス＋キーボード（タッチ非対応） |
| 対応フォーマット | `chart.json`（現行ゲームスキーマと完全互換） |
| ウィンドウサイズ | 1280×720 以上（リサイズ対応） |

### 1.3 サポートするノーツ種別

| type 値 | 概要 | 必須フィールド |
|---------|------|----------------|
| `normal` | 反射型・レーンなし | `time` |
| `top` | 反射型・Top判定ライン | `time`, `top_lane` (0〜2) |
| `vertical` | 垂直落下型 | `time`, `lane` (0〜6) |
| `long_normal` | ロング反射型・レーンなし | `time`, `end_time` |
| `long_top` | ロング反射型・Top判定ライン | `time`, `top_lane`, `end_time` |
| `long_vertical` | ロング垂直落下型 | `time`, `lane`, `end_time` |
| `chain` | チェインノーツ | `time`, `chain_type`, `chain_count`, `chain_interval`、型依存フィールド |

#### chain ノーツのフィールド

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| `chain_type` | string | ✓ | `"normal"` / `"top"` / `"vertical"` |
| `chain_count` | int | ✓ | メンバー総数（≥2） |
| `chain_interval` | float | ✓ | メンバー間の判定間隔（秒） |
| `top_lane` | int | chain_type="top" のみ | Top判定ライン (0〜2) |
| `lane` | int | chain_type="vertical" のみ | 下部判定レーン (0〜6) |
| `last_long` | bool | 任意 | 最後のメンバーをロングノーツにする場合 true |
| `last_end_time` | float | last_long=true のみ | 最後のメンバーの end_time（秒） |

## 2. 画面レイアウト

```
┌──────────────────────────────────────────────────────────────────┐
│  メニューバー  [File] [Edit] [View]                               │
├──────────────────────────────────────────────────────────────────┤
│  コントロールバー（再生・BPM・スナップ・ノーツ種別選択）           │
├────────────┬─────────────────────────────────────┬───────────────┤
│            │                                     │               │
│ トラック   │          タイムラインエリア          │  プロパティ   │
│ ヘッダー   │                                     │  パネル       │
│  (120px)   │           （メイン編集領域）         │   (240px)     │
│            │                                     │               │
├────────────┴─────────────────────────────────────┴───────────────┤
│  ステータスバー（カーソル時刻・総ノーツ数・ファイルパス等）        │
└──────────────────────────────────────────────────────────────────┘
```

### 2.1 コントロールバー

左から順に以下を配置する。

| コントロール | 内容 |
|-------------|------|
| 再生ボタン（▶） | 再生 / 一時停止トグル |
| 停止ボタン（■） | 停止してプレイヘッドを先頭に戻す |
| 時刻表示 | `0:00.000` 形式でプレイヘッド位置を表示 |
| BPM 入力 | 基準 BPM（float）。メタデータの `bpm` に対応 |
| スナップ分割セレクター | グリッドスナップの分割数を選択（後述） |
| スナップ ON/OFF トグル | グリッドスナップの有効・無効切り替え |
| ノーツ種別ツールバー | 現在配置するノーツの種別を選択（7種＋chain） |
| オフセット入力 | `meta.offset`（秒）を入力 |

### 2.2 トラックヘッダー

タイムラインの左端に各トラック行のラベルを表示する。
ヘッダーをクリックするとそのトラック全体のノーツを選択できる。

| 行グループ | 行ラベル | 背景色（目安） |
|-----------|---------|----------------|
| TOP グループ | `TOP 0` / `TOP 1` / `TOP 2` | オレンジ系 |
| NORMAL グループ | `NORMAL` | 青系 |
| VERTICAL グループ | `V 0` 〜 `V 6` | 青系（やや暗め） |

合計 11 行。グループ間にセパレーター行（高さ 4px）を挿入する。

### 2.3 タイムラインエリア

- X 軸: 時間（秒）。左端から右方向へ進む。
- Y 軸: トラック行（2.2 節の順序）。
- タイムラインエリア上部に時間ルーラーを表示する。
- プレイヘッド（縦線）が現在の再生位置を示す。
- タイムラインエリアはマウスホイールで水平スクロール、Ctrl+ホイールで水平ズーム。

### 2.4 プロパティパネル

選択中のノーツの属性を表示・編集するパネル。
複数選択時は共通フィールドのみ編集可能（個別フィールドは「複数」と表示）。

表示するフィールド:

| フィールド | 編集可否 | 備考 |
|-----------|---------|------|
| type | 読み取り専用 | ツールバーで変更 |
| time（秒） | ✓ | 数値入力 |
| end_time（秒） | ✓ | ロング系のみ表示 |
| top_lane | ✓ | top 系のみ（0〜2） |
| lane | ✓ | vertical 系のみ（0〜6） |
| chain_type | ✓ | chain のみ |
| chain_count | ✓ | chain のみ（≥2） |
| chain_interval | ✓ | chain のみ（秒） |
| last_long | ✓ | chain のみ（チェックボックス） |
| last_end_time | ✓ | chain かつ last_long=true のみ |

## 3. タイムライン仕様

### 3.1 ズームと表示幅

- **デフォルトズーム**: 1秒 = 200px
- **ズーム範囲**: 1秒 = 50px 〜 1秒 = 2000px
- **ズーム操作**: Ctrl+マウスホイール（プレイヘッド位置を中心に拡縮）
- **スクロール操作**: マウスホイール（水平）、またはスクロールバー

### 3.2 ルーラー表示

タイムラインエリア上部に時間ルーラーを表示する。

- BPM スナップ ON 時: 小節線（太線）・拍線（細線）・細分割線（薄線）を表示
- BPM スナップ OFF 時: 0.5 秒ごとに目盛りを表示
- ズームレベルに応じて表示密度を自動調整し、重なりを防ぐ

### 3.3 トラック行の高さ

| 状態 | 高さ |
|------|------|
| 通常 | 32px |
| グループセパレーター | 4px |

タイムラインエリアの縦方向スクロールは行数が多い場合（将来対応）に備えてサポートする。現行では合計 11 行 + セパレーターなので固定表示も可。

### 3.4 ノーツの描画

#### 通常ノーツ（normal / top / vertical）

- 幅: スナップ 1 グリッド幅（最小 8px）
- 高さ: トラック行高さの 80%
- 中心 X: `time` に対応するピクセル位置

#### ロングノーツ（long_normal / long_top / long_vertical）

- `time` から `end_time` までの横長の帯として描画
- 帯の左端が `time`、右端が `end_time`
- 帯の高さ: トラック行高さの 60%
- 両端に丸い頭部を描画

#### chain ノーツ

- `chain_count` 個のノーツが `chain_interval` 間隔で並んで表示される
- 同一トラック行内に配置され、メンバー間をコネクターライン（細線）で結ぶ
- `last_long = true` の場合、最後のメンバーを帯として描画する
- chain 全体を選択するとグループとして扱う（メンバー個別選択は Ctrl クリック）

#### ノーツの色

| 種別 | 色 |
|------|-----|
| normal / long_normal | 青（`#4D66FF`） |
| top / long_top | 緑（`#1AE64D`） |
| vertical / long_vertical | 紫（`#8855FF`） |
| chain（normal系） | 青系・明度高め |
| chain（top系） | 緑系・明度高め |
| chain（vertical系） | 紫系・明度高め |
| 選択中 | 黄色アウトライン付き |

## 4. ノーツ配置操作

### 4.1 ツール

コントロールバーのノーツ種別ツールバーで配置するノーツの種別を選択する。
選択可能な種別: `normal` / `top` / `vertical` / `long_normal` / `long_top` / `long_vertical` / `chain`

数字キー（1〜7）で素早く切り替えられる（後述ショートカット参照）。

### 4.2 ノーツの配置

#### 単発ノーツ（normal / top / vertical）

1. ツールバーで種別を選択する
2. タイムライン上の対象トラック行の任意の位置を**左クリック**する
3. スナップが ON の場合、最も近いグリッド位置にスナップして配置される
4. クリック位置の X 座標が `time` として設定される

#### ロングノーツ（long_normal / long_top / long_vertical）

1. ツールバーで種別を選択する
2. 対象トラック行の `time` となる位置で**左ボタンを押し**、そのまま右方向へ**ドラッグ**して `end_time` となる位置で**離す**
3. ドラッグ中はリアルタイムに帯が描画される
4. 最小長: 1 グリッド幅。それ未満は配置しない

#### chain ノーツ

1. ツールバーで `chain` を選択する
2. 対象トラック行の `time` となる位置を**左クリック**する
3. クリック直後にプロパティパネルが開かれ、以下の設定を求める:
   - `chain_count`（デフォルト: 3）
   - `chain_interval`（デフォルト: 0.4 秒）
   - `last_long`（デフォルト: false）
   - `last_end_time`（last_long=true の場合）
4. 確定するとタイムラインに chain が配置される

### 4.3 ノーツの削除

- 右クリック: クリックしたノーツを削除
- Delete キー: 選択中のノーツをすべて削除

### 4.4 ノーツの選択

| 操作 | 結果 |
|------|------|
| 左クリック（配置ツール以外） | 単一選択 |
| Ctrl + 左クリック | 選択に追加 / 解除 |
| タイムライン上でドラッグ（何もない場所） | 矩形選択 |
| Ctrl + A | 全選択 |
| Escape | 選択解除 |

**選択ツールモード（S キー）**: このモードではクリックしてもノーツは配置されず、選択・移動のみ行う。

### 4.5 ノーツの移動

選択ツールモードで選択中のノーツを左ドラッグで移動する。

- 水平方向（時間軸）: スナップが ON の場合グリッドにスナップ
- 垂直方向（トラック行）: 対象の行に移動する。ノーツ種別が変わる移動（例: TOP行→VERTICAL行）は**禁止**。同種グループ内の行変換（例: TOP 0 → TOP 2）は許可する。
- chain ノーツ: グループ全体が一緒に移動する

### 4.6 ノーツの複製

- Ctrl + D: 選択中のノーツを複製して、元のノーツの 1 グリッド右に配置する
- Ctrl + C / Ctrl + V: クリップボードへのコピー＆貼り付け

### 4.7 Undo / Redo

すべての編集操作（配置・削除・移動）は Undo/Redo スタックに積まれる。

- Ctrl + Z: Undo（無制限）
- Ctrl + Y または Ctrl + Shift + Z: Redo

## 5. 再生コントロール

### 5.1 基本操作

| 操作 | 動作 |
|------|------|
| Space | 再生 / 一時停止のトグル |
| Escape（再生中） | 停止（プレイヘッドを再生開始位置に戻す） |
| Home | プレイヘッドを先頭（0秒）に移動 |
| End | プレイヘッドを末尾に移動 |
| ← / → | プレイヘッドを 1 グリッド分移動 |
| Shift + ← / → | プレイヘッドを 1 小節分移動 |

### 5.2 プレイヘッド操作

- タイムラインのルーラー部分を左クリックするとプレイヘッドをその位置に移動する
- 再生中にルーラーをクリックした場合、その位置から再生を再開する

### 5.3 再生中の表示

- 再生中はプレイヘッド（縦線）がリアルタイムに移動する
- タイムラインは自動スクロールし、プレイヘッドが常に画面内（右側 20% 付近）に表示される
- **再生中もノーツの配置・削除が可能**（ただし Undo できるのは停止後）

### 5.4 音楽ファイルの読み込み

- メタデータパネルまたはメニューから音楽ファイル（OGG / WAV）を指定する
- 指定したファイルパスはメタデータの `audio` フィールドに書き込まれる
- 再生開始時に `meta.offset` 秒だけ遅延させてから再生する（offset が正の場合、offset 秒後から音楽を再生する）

## 6. BPMグリッドとスナップ

### 6.1 グリッドスナップの目的

BPM に基づいた拍・小節の格子へノーツ配置位置を自動的に丸める機能。
音楽のリズムに合わせた正確な譜面作成を支援する。

### 6.2 スナップ分割設定

コントロールバーのセレクターで以下から選択する。

| 分割値 | 意味 | グリッド間隔（BPM 120 の場合） |
|--------|------|-------------------------------|
| 1 | 4分音符（1拍）単位 | 0.500 秒 |
| 2 | 8分音符（1/2拍）単位 | 0.250 秒 |
| 3 | 3連符（1/3拍）単位 | 0.167 秒 |
| 4 | 16分音符（1/4拍）単位 | 0.125 秒 |
| 6 | 3連16分（1/6拍）単位 | 0.083 秒 |
| 8 | 32分音符（1/8拍）単位 | 0.063 秒 |

スナップが ON の場合、ノーツの `time`・`end_time` はグリッド位置に丸められる。

### 6.3 スナップ計算式

```
beat_duration = 60.0 / bpm_at(time)    // その時刻のBPMから1拍の長さを算出
grid_interval = beat_duration / snap_division
snapped_time = round(time / grid_interval) * grid_interval
```

BPM チェンジがある場合、`bpm_at(time)` で現在のBPMを参照する（第7節参照）。

### 6.4 グリッド線の表示

タイムライン上にグリッド線を表示する。

| 線の種類 | 条件 | 表示 |
|---------|------|------|
| 小節線 | 常に（BPMが設定されている場合） | 太い白線（アルファ 0.5） |
| 拍線 | 常に | 細い白線（アルファ 0.3） |
| 細分割線 | スナップ分割 ≥ 2 | 薄い白線（アルファ 0.15） |

ズームレベルが低い場合（グリッド間隔が 4px 未満になる場合）、細分割線を非表示にする。

## 7. BPM変化（BPMチェンジ）

### 7.1 概要

`meta.bpm_changes` 配列を編集する機能。
曲中に BPM が変化する場合に使用する。

```json
"bpm_changes": [
  {"time": 0.0, "bpm": 120.0},
  {"time": 20.0, "bpm": 150.0}
]
```

- `time=0.0` のエントリは必須（先頭の基準 BPM）
- `meta.bpm` は `bpm_changes[0].bpm` と同値に保つ
- `bpm_changes` が空の場合、`meta.bpm` の値を `[{"time": 0.0, "bpm": <meta.bpm>}]` として扱う

### 7.2 BPMチェンジの編集操作

- **BPMチェンジマーカー**: タイムラインのルーラー直下に専用の帯（高さ 16px）を設ける
- BPM チェンジが存在する時刻にはマーカー（縦線＋BPM値テキスト）を表示する
- **追加**: Ctrl + B キーを押すか、帯上の任意の位置を右クリック → 「BPMチェンジを追加」
  - 追加時にダイアログで BPM 値を入力する
- **編集**: マーカーを左クリックして選択し、プロパティパネルで BPM 値・時刻を編集する
- **削除**: マーカーを選択して Delete キー（time=0.0 のエントリは削除不可）
- **移動**: マーカーをドラッグして時刻を変更する（time=0.0 は移動不可）

## 8. ファイル操作

### 8.1 新規作成

メニュー「File → New」で新規の chart.json を作成する。
以下のデフォルト値でメタデータを初期化する。

```json
{
  "meta": {
    "title": "",
    "artist": "",
    "level": 1,
    "bpm": 120.0,
    "offset": 0.0,
    "audio": "",
    "bpm_changes": [{"time": 0.0, "bpm": 120.0}]
  },
  "notes": []
}
```

### 8.2 ファイルを開く

メニュー「File → Open」（Ctrl + O）で既存の `chart.json` を開く。

- ファイルダイアログで `chart.json` を選択する
- 読み込み後、同一フォルダ内に `meta.audio` で指定された音楽ファイルが存在する場合、自動的に読み込む
- 不明なフィールドは無視するが、保存時にも保持する（将来互換性のため）
- `bpm_changes` が存在しない場合、`meta.bpm` から自動生成する

### 8.3 上書き保存

メニュー「File → Save」（Ctrl + S）で現在の `chart.json` に上書き保存する。

- 保存形式: インデントあり JSON（読みやすさのため）
- notes 配列は `time` の昇順でソートして保存する
- `meta.bpm` は `bpm_changes[0].bpm` と同値で保存する

### 8.4 名前を付けて保存

メニュー「File → Save As」（Ctrl + Shift + S）でファイルダイアログを開いて保存先を指定する。

### 8.5 未保存変更の警告

未保存の変更がある状態でファイルを閉じる・新規作成・終了しようとした場合、
「保存しますか？（はい / いいえ / キャンセル）」のダイアログを表示する。

## 9. メタデータ編集

プロパティパネルの「メタデータ」セクション、またはメニュー「Edit → Metadata」で以下を編集する。

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `title` | string | 楽曲名 |
| `artist` | string | アーティスト名 |
| `level` | int (1〜10) | 難易度レベル |
| `bpm` | float | 基準 BPM（bpm_changes[0] と連動） |
| `offset` | float (秒) | 音声開始から最初の拍までのオフセット |
| `audio` | string | 音声ファイル名（ファイルブラウザで選択可） |

`offset` の調整補助:

- 「オフセット調整モード」ボタンを押すと、曲を再生しながら任意のタイミングでスペースキーを押すと最初の拍として offset を自動設定する（タップテンポ方式）。

### ノーツ種別選択

### 11.4 時刻の内部表現

- すべての時刻は **秒（float）** で保持する
- 描画時に `pixel_x = (time - scroll_offset) * pixels_per_second` に変換する
- ピクセルから時刻への変換: `time = pixel_x / pixels_per_second + scroll_offset`

### 11.5 Undo/Redo 実装方針

Command パターンで実装する。各操作は `execute()` と `undo()` を持つ。
対象操作: ノーツの追加・削除・移動・プロパティ変更・BPMチェンジの追加・削除・変更。

### 11.6 chart.json の保存フォーマット

ゲーム本体との互換性を保つため、以下のルールで保存する。

```json
{
  "meta": {
    "title": "...",
    "artist": "...",
    "level": 5,
    "bpm": 120.0,
    "offset": 0.0,
    "audio": "music.ogg",
    "bpm_changes": [{"time": 0.0, "bpm": 120.0}]
  },
  "notes": [
    // time の昇順でソート
  ]
}
```

- `normal` ノーツは `lane` フィールドを**含まない**（ゲーム側でランダム決定するため）
- `top` / `long_top` ノーツは `top_lane` フィールドを含む
- `vertical` / `long_vertical` ノーツは `lane` フィールドを含む
- ロング系ノーツは `end_time` フィールドを含む
- `chain` ノーツは該当フィールドをすべて含む

### 11.7 スコープ外

本バージョン（v1.0）では以下を対象外とする。

- プレビュー再生（ゲーム画面と同様にノーツが落下するプレビュー）
- 複数難易度の同時編集
- オートセーブ
- プラグイン・拡張機能
